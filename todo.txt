[MEDIUM] blog_posts.published_at DEFAULT NOW() on creation
  Drafts (is_published=false) get published_at = creation time, not publish time.
  Fix: published_at is NULL by default, set when is_published flips to true.
  Requires backend logic to set it on first publish.

[MEDIUM] No updated_at trigger on blog_posts
  updated_at must be set manually by app. Will drift if any direct DB update happens.
  Add trigger:
    CREATE OR REPLACE FUNCTION update_updated_at()
    RETURNS TRIGGER AS $$ BEGIN NEW.updated_at = NOW(); RETURN NEW; END; $$ LANGUAGE plpgsql;
    CREATE TRIGGER trg_blog_posts_updated_at
    BEFORE UPDATE ON blog_posts FOR EACH ROW EXECUTE FUNCTION update_updated_at();

[MEDIUM] No updated_at on products table
  No way to know when a product was last modified. Useful for cache invalidation, admin audit.
  Add: updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW() with same trigger pattern.

[MEDIUM] TIMESTAMP instead of TIMESTAMPTZ
  images.created_at, blog_posts.published_at, blog_posts.updated_at all TIMESTAMP (no tz).
  Server timezone change or future internationalisation will cause silent data corruption.
  Migrate all timestamp columns to TIMESTAMPTZ.
