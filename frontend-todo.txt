MOBILE UX (Consolidated)
================================================================================

This section consolidates all mobile-specific findings from the UX audit and
the project todo.txt into one place. Items referenced in other sections above
are cross-referenced but not removed -- this section is the authoritative
mobile checklist.

[MOB1] Full Mobile Redesign Needed (from todo.txt)
-------------------------------------------------------------------------------
Current state: media queries exist but layout is desktop-first and janky on
mobile browsers. Full responsive redesign needed, not just breakpoint patches.

Approach:
  - Audit with Chrome DevTools: 375px (iPhone SE), 390px (iPhone 14), 768px
  - Switch to mobile-first CSS (min-width breakpoints instead of max-width)
  - Use CSS container queries for component-level responsiveness
  - Test on real devices, not just browser emulation

[MOB2] Navigation & Header
-------------------------------------------------------------------------------
- Hamburger menu missing aria-expanded, aria-label -- unusable for screen
  readers on mobile (see also C3)
  File: frontend/src/App.tsx (line 38)
- Sticky nav height eats screen space on small phones
  File: frontend/src/App.css
- Main nav positioned off-screen with left: -100% -- focus trap risk; keyboard
  users can tab into invisible nav
  File: frontend/src/App.css (line 142)
- Logo image jumps between fixed heights (90px, 65px, 55px) with no
  intermediate tablet size
  File: frontend/src/App.css
- No skip-to-content link for keyboard/mobile screen reader users (see also C3)

[MOB3] Shop Page -- Filters & Grid
-------------------------------------------------------------------------------
- 9 filter groups stack vertically on mobile -- excessive scrolling, no
  collapse/expand toggle. Need overlay/drawer pattern instead of sidebar.
  File: frontend/src/pages/Shop.tsx (lines 65-253)
  File: frontend/src/pages/Shop.css (lines 209-233)
- Sidebar hidden at 992px but grid layout remains 1fr with no mobile
  optimization
- No filter count badge to show active filters at a glance on mobile
- Product grid should go single-column on small screens; card sizing and image
  aspect ratio need tuning
- No "No products found" empty state when filters match nothing -- especially
  confusing on mobile with collapsed sidebar (see also H3)

[MOB4] Product Details
-------------------------------------------------------------------------------
- Quantity selector buttons too small to tap on mobile -- need >= 44x44px touch
  targets
  File: frontend/src/pages/ProductDetails.tsx (lines 167-173)
- Product attribute table needs mobile-friendly layout (card view or stacked)
  File: frontend/src/pages/ProductDetails.tsx
- Add-to-cart button sizing needs mobile treatment
- Product detail image hover scale (1.01) has no transition defined -- janky
  on touch devices
  File: frontend/src/pages/ProductDetails.css (lines 50-52)
- Missing 1024px tablet breakpoint -- jumps from 992px to 768px
  File: frontend/src/pages/ProductDetails.css (lines 203-240)

[MOB5] Cart Page
-------------------------------------------------------------------------------
- Cart table layout breaks on narrow screens -- needs card-based mobile layout
  File: frontend/src/pages/Cart.tsx
- Quantity +/- buttons same tap target issue as ProductDetails
  File: frontend/src/pages/Cart.tsx (lines 44-56)
- Cart summary sticky positioning has no transition when switching to static
  at breakpoint
  File: frontend/src/pages/Cart.css (lines 157-164)

[MOB6] Blog
-------------------------------------------------------------------------------
- ReactMarkdown table rendering overflows on mobile -- tables not responsive
  File: frontend/src/pages/Blog.tsx (lines 109-128)
  File: frontend/src/pages/BlogPostDetail.tsx (lines 93-113)
- Blog content width, typography scale, image sizing need mobile treatment
- Only 768px and 480px breakpoints; missing 1024px tablet optimization
  File: frontend/src/pages/Blog.css (lines 254-280)
- Blog post cards have no max-width constraints on mobile

[MOB7] Admin Panel -- Completely Unusable on Mobile
-------------------------------------------------------------------------------
- Admin sidebar has no mobile hamburger -- no way to navigate on phones
  File: frontend/src/pages/admin/AdminLayout.tsx
  File: frontend/src/pages/admin/Admin.css (line 1325)
- Admin sidebar only has responsive rules at 992px and 768px, no intermediate
  tablet breakpoint
- Products table overflows on small screens -- no horizontal scroll indicator,
  action buttons not mobile-sized, thumbnails tiny
  File: frontend/src/pages/admin/AdminProducts.tsx (lines 183-274)
- Admin dashboard grid jumps from repeat(auto-fit, minmax(240px, 1fr)) to full
  reflow at 992px with no intermediate state
  File: frontend/src/pages/admin/Admin.css (lines 1325-1444)
- BlogForm two 10-row textareas side-by-side -- unusable on mobile, should
  stack responsively
  File: frontend/src/pages/admin/BlogForm.tsx (lines 288-300)
- Admin image grid not responsive at different breakpoints; upload section and
  images section both full-width
  File: frontend/src/pages/admin/AdminImages.tsx
- Admin forms: input fields, date picker, image selector all need mobile
  treatment
- Admin stat cards loading state shows '...' -- even worse on small screens
  where layout shifts

[MOB8] Touch Targets & Tap Sizing
-------------------------------------------------------------------------------
- Pagination buttons too small for mobile thumbs -- need >= 44x44px
  File: frontend/src/components/Pagination.tsx
- Quantity +/- buttons throughout (ProductDetails, Cart)
- Admin action buttons (edit/delete) in product table
- Shop filter checkboxes -- default browser checkbox size too small for touch
- CollapsibleSection toggle button needs adequate tap area

[MOB9] Images & Performance on Mobile
-------------------------------------------------------------------------------
- No lazy loading on product images -- wastes mobile data
  File: frontend/src/components/ProductCard.tsx
- No responsive srcset -- oversized images sent to mobile devices
- No image error fallback -- broken images worse on mobile (no retry)
- No skeleton/placeholder while images load -- layout shift on slow connections
- Hero logo jumps from 300px to 180px at 768px with no intermediate size
  File: frontend/src/pages/Home.css

[MOB10] Typography & Spacing on Mobile
-------------------------------------------------------------------------------
- Font sizes hardcoded with media queries instead of clamp() -- no fluid
  scaling between breakpoints (see also M3)
- Heading sizes inconsistent at mobile breakpoints:
  - h1 jumps between 3rem and 2rem with no intermediate
  - h2 varies between 2.5rem and 1.5rem across pages
- Line heights and spacing scale not optimized for small screens
- Form labels at 0.95rem may be too small on mobile

[MOB11] Breakpoint Architecture
-------------------------------------------------------------------------------
- Desktop-first approach (max-width queries) throughout -- should be
  mobile-first (min-width)
  Files: App.css (768px, 480px), Shop.css (992px, 768px), Cart.css (992px,
  768px), Home.css (768px only), ProductDetails.css (992px, 768px),
  Blog.css (768px, 480px), Admin.css (992px, 768px)
- Missing 1024px tablet landscape breakpoint across all pages
- No consistent breakpoint scale defined as CSS variables
- Recommended scale: 480px (small phone), 640px (large phone), 768px (tablet
  portrait), 1024px (tablet landscape), 1280px (desktop)
